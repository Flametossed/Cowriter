# trope-check

Flag the unexamined defaults. Scan against [trope-slop.md](../../../frameworks/craft/trope-slop.md).

1. Take the premise / plot spine / beat sketch as input.
2. Check each element against the kill-list categories (premise spines, plot turns, villain/character defaults, setting tells, naming & logline tells).
3. Apply the **two gates** before flagging: **awareness** (did the writer name it as the default?) and **angle** (is one axis genuinely twisted?). Authority order: `writer's intent > fresh angle > trope-slop`. Flag only elements that pass *neither* gate.
4. Report per the framework's format: `default → gates → opening`. Give an *opening*, not a verdict — the writer owns authorship.
5. End with an originality read: how many load-bearing elements are unexamined defaults + the single highest-leverage twist. If it fails many markers, route back to `twist`.

> For *prose* slop (not idea slop) in actual drafts, that's Cowriter's `slop-check`, not this.
