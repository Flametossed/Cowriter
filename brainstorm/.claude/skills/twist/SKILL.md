# twist

Make it original. Follow [twist-method.md](../../../frameworks/ideation/twist-method.md).

1. **Name the default first** — write the version a model or the market would produce by reflex. That's the target to move away from. Run `trope-check` to surface defaults it's leaning on.
2. **Pick ONE subversion lever** — invert the trope / move the camera / change the engine-cost / relocate / literalize. One axis of weird, not five.
3. **Protect the heat** — name why the writer picked this spark; the twist must keep that.
4. **Specificity pass** — swap every generic noun for a load-bearing specific one.
5. **Calibrate on the familiarity dial** — one familiar anchor + one clear twist. Show, steer if disliked, log the default named + lever used + the twisted concept.
