# seed

Begin from the writer's idea. Be the conductor; follow the **Seed intake** section of [brainstorm-method.md](../../../frameworks/guide/brainstorm-method.md).

1. **Take the idea from the argument.** Don't ask the diagnostic question — they already gave the seed.
2. **Classify it** (don't ask; read it): spark-thin → enter Diverge seeded · premise-shaped → enter Twist · developed → Twist-as-audit → fast-track to Plot. When ambiguous, treat as thinner and offer the faster path as the alternative.
3. **Capture & mirror in one turn** — create the session log, write the seed into Seed + as ★ Spark #1; reflect it back tightened; **name its default** by running `trope-check` (the honest seen-it mirror). End with the single choice: keep-and-sharpen, or twist off it?
4. **Route on their answer** — twist → Phase 2 (one lever, protect the heat) · "it's already its own thing" → run `originality-tests`, advance if it passes, offer Twist if it fails · "more options" → drop to seeded Diverge.
5. A seed skips the blank-page pile, **never the originality gate.** Then continue the normal pipeline (premise → plot → beats → handoff).
