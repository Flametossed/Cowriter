# Originality Tests — the pressure battery

> Portable. A battery of fast tests to run on a premise, plot, or beat before it locks. Where `trope-slop` catches *specific* default moves, these tests catch *generic-ness* in the round. Used by the conductor in Phase 3 and any time the writer asks "is this original enough?"

> Originality is not novelty-for-its-own-sake. The goal of these tests is not "maximize strange" — it's "is there a real choice here, or did the concept settle to the center by default?" A premise can pass every test and still be conventional *on purpose*; what fails is the *unintended* default.

---

## Run order

Cheap-and-decisive first. Stop early if one fails hard — fix that, then re-run.

### 1. The "seen it" test
Name **three existing works** this premise resembles. If you can name them in five seconds *and your version adds nothing they don't have*, it's derivative. If you can't name even one, either it's genuinely fresh or you don't know the field (check the second possibility honestly).
- **Pass:** you can name comps, but yours has a clear axis they lack.
- **Fail:** the comps *are* the pitch. → back to `twist-method`.

### 2. The logline-swap test
Could this exact logline headline a dozen other books? Strip the proper nouns and read it. *"A young person discovers they have power and must save their world"* describes a thousand books — the logline is centered.
- **Fix:** the load-bearing specifics are missing from the logline. Push them up into it (`twist-method` specificity pass).

### 3. The inversion test
Flip the premise's main axis and ask: *is the flipped version more interesting than the original?* If yes — **write the flipped one.** This test routinely finds the better book hiding behind the default.
- Example: "a detective hunts a killer" → "a killer hunts the one detective who can clear them." If the flip lights up, follow it.

### 4. The specificity test
Count the **generic nouns** carrying weight in the premise: *a hero, a kingdom, a dark force, a power, a chosen one, a journey.* Each one is a choice not yet made.
- **Pass:** the key nouns are particular and load-bearing (a *debt-citizenship city*, a *memory-priced magic*).
- **Fail:** three or more generic placeholders → the concept is a template with blanks. Fill them (`twist-method`).

### 5. The "why this character / why now" test
Could you swap the protagonist for a generic competent person and keep the same plot? Could it happen a decade earlier or later with no change? If yes, the protagonist and the moment are *interchangeable* — a sign the plot is bolted on, not grown.
- **Pass:** only *this* person, *now*, generates *this* story — their specific flaw/history is the engine's fuel.

### 6. The engine test (inherited from Cowriter's premise method)
Does the premise *generate scenes*, or only *describe a situation*? Try to name the next three things that happen. If you can't — if it's a static condition ("a world where X") with no built-in pressure — it's a setting, not a story.
- **Fail:** no engine → `plot-method` (or back to `twist` if the premise itself is inert).

### 7. The familiarity-dial read (calibration, not pass/fail)
Locate the concept on the dial from `twist-method`:
- **Too far familiar** (no nameable twist) → it's slop; add one strong axis.
- **Too far novel** (no nameable anchor, two readers can't agree what kind of story it is) → re-anchor in one recognizable promise.
- **Right:** one clear anchor + one clear twist. Readers know what they're getting *and* what's new.

---

## Reporting

Give a short verdict per test (pass / fail / borderline) and **one** recommended next move — never a wall of criticism. The output is always a *direction*, not a grade:

```
seen-it: borderline — comps: <A, B>. Your axis (<X>) is real but soft.
specificity: fail — "a kingdom", "a dark power" are placeholders.
engine: pass — debt-citizenship forces foreclosure scenes.
→ Highest-leverage fix: specify the kingdom into the debt-city; the rest follows.
```

If a concept fails 1, 2, or 6, **do not advance** to plot/beats/handoff — the spine isn't sound yet. Route back to Twist or Spark. Passing these isn't about being weird; it's about having *made the choices* instead of letting the model's center make them for you.
