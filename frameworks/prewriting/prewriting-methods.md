# Pre-writing Methods — premise, character, world

> Portable. Three pre-writing methods that fill the bible. Each section stands alone — paste just the one you need. Outlining has its own file ([outline-method.md](outline-method.md)).

> All three **write to the bible** and **read what's already there** — never contradict locked canon, and mirror checkable facts into `canon-facts.md`.

---

## premise

Sharpen the book's spine. Writes `bible/premise.md`.

1. Start from the user's seed (a line, a vibe, a "what if"). Ask only what's missing.
2. Drive to a **logline** in the form: *[protagonist] must [goal] or [stakes], but [obstacle].* If it won't compress, the idea isn't sharp — keep pushing.
3. Derive theme as a **question**, not a moral. The book argues; it doesn't preach.
4. Split conflict three ways: external (plot) / internal (arc) / philosophical (value vs counter-value).
5. Pin tone + comps ("X meets Y") and a **promise to the reader** — the experience opening the book guarantees.
6. Force a **"what this is NOT"** list. Guardrails prevent drift more than goals do.

Pressure-test: does the premise *generate scenes*? If it describes a situation but implies no engine of conflict, it's a setting, not a premise. Flag that.

---

## character

Build a character the plot can use. Appends to `bible/characters.md`, mirrors locked details to `canon-facts.md`.

1. Anchor to the story's need: what role does the plot require? Don't build a person, then strand them.
2. Core engine — **Want / Need / Lie / Ghost**:
   - Want = conscious goal (drives plot).
   - Need = truth they must learn (drives arc).
   - Lie = false belief blocking the need.
   - Ghost = the wound that planted the lie.
   The arc is the Lie breaking under pressure until the Need wins (or tragically doesn't).
3. **Voice profile** — rhythm, vocabulary, tics, what they refuse to say, one sample line only they'd speak. This is what `dialogue` and `scene` enforce.
4. **Relationships** — define by tension, not bio. Who do they clash with and why?
5. **Locked canon** — age, physical markers (eye color, scars, height), background facts → mirror to `canon-facts.md` so `continuity` can check prose against them.

Pressure-test: state the character's arc in one sentence (start state → turn → end state). If you can't, the arc isn't real yet.

---

## world

Build setting that serves story, not setting for its own sake. Writes `bible/world.md`, mirrors facts to `canon-facts.md`.

1. **Constrain, don't catalogue.** Record only what affects plot or could create contradictions. A magic/tech/social system needs its **limit and cost** stated, not just its power — limits create tension.
2. **Locations** — for each recurring place, fix its *sensory signature* (the 2–3 details that make it itself) and its hard canon (layout, distances) so geography stays consistent.
3. **Factions** — define by goal + method + who they conflict with.
4. **Social texture** — power, money, customs, taboos, invented terms (lock spellings in `canon-facts.md`).
5. **World-level timeline anchors** — big historical events the plot leans on (per-scene chronology lives in `timeline.md`).

Pressure-test: for every world element, ask "does a scene depend on this?" If no, cut it from the bible (keep it in a notes file if you love it). Worldbuilding bloat is procrastination.
